FAQ
How does random number generation work?
Each block in the Ethereum blockchain has a verification hash. The Ethereum Virtual Machine allows you to retrieve these hashes using the blockhash() function. The function receives a numeric argument with the block number as input. As an entropy for the RNG, we use the hash of the future block, which is unknown at the time of the transaction. The attacker will not be able to create an exploit contract with the same RNG code and determine the result in advance, because the result of the bet will be known only for the next block from the moment of the transaction.
How it works?
Step by step
  • The player bets. The contract remembers the block number of its transaction
  • The next time you call the contract, the result of the bet is determined on the basis of the check block hash of the bet, and if the bet turns out to be winning, the winner is sent the winning ethers
  • Our bot automatically analyzes all transactions and quickly sends a transaction to the contract to complete the bet.
