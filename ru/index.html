FAQ
Как работает генерация случайных чисел?
У каждого блока в цепочке Ethereum есть проверочный хэш. Виртуальная машина Ethereum Virtual Machine позволяет получать эти хэши с помощью функции blockhash(). Функция получает на вход числовой аргумент с указанием номера блока. В качестве энтропии для ГПСЧ мы используем хэш будущего блока, который неизвестен на момент создания транзакции. Злоумышленник не сможет создать эксплоит-контракт с тем же кодом ГПСЧ и заранее определить результат, ибо результат ставки будет известен только на следующий блок с момента совершения транзакции.
Объясните, как это работает на примере
Сценарий работы смарт-контракта реализован следующим образом:
  • Игрок делает ставку. Контракт запоминает номер блока его транзакции
  • При любом следующем обращении к контракту, результат ставки определяется на основе проверочного хеша блока ставки, и, если ставка оказывается выигрышной, победителю отправляются его победные эфиры
  • Наш бот автоматически анализирует все входящие транзакции контракта и практически моментально подводит итоги ставок игроков
